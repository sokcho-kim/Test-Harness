## 4. 프롬프트 작성 영역 (메인)

### 4.1 상단 컨트롤

| 요소 | 현재 값 | 설명 |
|------|---------|------|
| **템플릿 선택** | 드롭다운 | 미리 저장된 템플릿 불러오기 |
| **모델 선택** | `gpt-4o-mini` | 테스트할 LLM 모델 |
| **+ 모델별 비교** | 버튼 (파란색) | 다중 모델 동시 테스트 |

### 4.2 메시지 구성

| 구성요소 | 설명 |
|----------|------|
| **SYSTEM** | 시스템 프롬프트 - `"You are a helpful assistant."` |
| **role** | 메시지 역할 선택 (system/user/assistant) |
| **message** | 실제 메시지 내용 입력 |

### 4.3 액션 버튼

| 버튼 | 기능 |
|------|------|
| **+ 메시지 추가** | 멀티턴 대화 구성 |
| **실행** | 프롬프트 실행 (숫자 1 = 실행 횟수?) |
| **메시지 비우기** | 입력 초기화 |

## 5. 변수 테스트 섹션

> "입력 변수가 없습니다.
> 프롬프트에 입력 변수를 {$variable} 형식으로 추가해주세요."

### 사용법 추정
```
SYSTEM: 당신은 {$role} 전문가입니다.
USER: {$question}에 대해 설명해주세요.
```

→ 변수별로 여러 값을 테스트할 수 있는 기능

---

## 6. 실행 옵션 (우측 패널)

| 파라미터 | 기본값 | 범위 | 설명 |
|----------|--------|------|------|
| **Temperature** | 1 | 0~2 | 출력 다양성 |
| **Max Tokens** | 2048 | - | 최대 출력 토큰 |
| **Stop Sequences** | (없음) | 텍스트 | 생성 중단 문자열 (콤마 구분) |
| **Top P** | 1 | 0~1 | 토큰 선택 범위 |
| **Frequency penalty** | 0 | 0~2 | 반복 단어 패널티 |
| **Presence penalty** | 0 | 0~2 | 반복 주제 패널티 |

---

## 7. 실행결과 영역

### 실행결과 로그
- 프롬프트 실행 후 응답 표시
- 현재: "아직 실행 결과가 없습니다."

### 실행 메트릭
- 토큰 사용량, 응답 시간 등 표시 추정
- 현재: "아직 실행 결과가 없습니다."

## 8. 핵심 기능 요약

| 기능 | 설명 | 용도 |
|------|------|------|
| **프롬프트 테스트** | 프롬프트 작성 → 즉시 실행 → 결과 확인 | 프롬프트 개발 |
| **모델 비교** | 동일 프롬프트를 여러 모델에서 테스트 | 모델 선택 |
| **변수 테스트** | {$var} 형식으로 여러 입력값 테스트 | A/B 테스트 |
| **파라미터 조정** | Temperature, Top P 등 실시간 조정 | 최적화 |
| **템플릿 저장** | 자주 쓰는 프롬프트 저장/불러오기 | 재사용 |
| **실행 로그** | 히스토리 및 메트릭 확인 | 분석 |
